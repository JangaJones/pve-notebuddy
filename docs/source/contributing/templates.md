# Contributing Templates

Template contributions are welcome.

## What Makes a Good Template

- clear service title
- valid website/fqdn info
- realistic default port (if known)
- useful default config path(s)
- stable icon URL

## Submission Tips

1. Base your JSON on existing templates for consistency.
2. Validate formatting and schema behavior in the UI.
3. Include evidence/source for defaults (official docs link if possible).
4. Keep fields concise and operationally useful.

## Where to Place Files

- add/update JSON in `templates/community-scripts/` or appropriate dataset folder
- ensure `templates/index.json` entry exists and is correct
